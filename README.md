# 阅卷辅助器 · BetterGrader

> 一个**单人、本地优先**的桌面程序,用来批改**奥赛纸质大题**。
> 价值锚只有一个:**键盘驱动的判分循环**——把翻页寻址、手眼往返录入、事后聚合这些非判断性损耗全部消灭,只留「看题—给分」一个动作,并让它在键盘上快到极致。

用 [Tauri v2](https://tauri.app/)(Rust 后端 + WebView2 前端)+ Vue 3 构建。所有数据留在本地,不联网;一场考试 = 一个可拎走的自包含目录。

## 它解决什么

奥赛判卷的真正耗时不在「判断给几分」,而在它周围的损耗:翻找这是谁的卷、在纸和录入框之间来回、批完再手动聚合排名。BetterGrader 把这些全部消灭:

- **全键盘横向批题** —— 选一道题,`1–9` 一键给档位分、`←/→` 上/下一份、`↑/↓` 速览同一考生别的页,几百份一道题连着批完。
- **两条导入路径,一个批法** —— 一堆扫描图(去「标注」逐页指认),或一叠按人命名的 PDF(每个 = 一个考生,导入即自动标注、直接开批)。
- **实时校准** —— 判分时左侧实时显示本题**分数分布直方图 + 平均分**,把「标准漂移」从批完才发现变成过程中自校正。
- **按题绑定的评分标准** —— 每题填一段 Markdown 参考答案,判分时按 `R` 随时对照。
- **三态 + 缺考,绝不静默填 0** —— 未判/存疑在成绩表里留空或标 `?`,缺考整行标「缺考」;**只有全部判完才输出排名**,绝不用带洞数据排出一个「像真的」名次。
- **导出** —— 成绩 CSV(Excel 直接认中文)、可打印报表、每人一份拼接好的 PDF。

## 快速开始

需要 [Rust](https://www.rust-lang.org/) + [Node.js](https://nodejs.org/) + [Tauri v2 环境依赖](https://tauri.app/start/prerequisites/)。

```bash
npm install
npm run tauri dev      # 开发运行
npm run tauri build    # 打包(见下方注意)
```

首次进入:「考试设置」→ 新建考试(选一个空目录)→ 建题 + 每题满分 + 判分档位 + 花名册 → 导入图片/PDF → 「判分」开批 → 「导出」出分。完整流程与**快捷键表**见 [使用手册](docs/03-使用手册.md)。

## 架构一览

三层刻意解耦:

- **`crates/grading-core/`** —— 纯领域逻辑(队列、三态、导出、标注计数…),不依赖 Tauri/前端,`cargo test -p grading-core` 独立测。
- **`src-tauri/src/commands.rs`** —— 薄 IPC 层,锁状态、转发,无业务逻辑。
- **`src/`** —— Vue 3 + TypeScript + naive-ui 前端;判分/标注的键盘逻辑是**无副作用纯 reducer**,Vitest 单测。

## 开发命令

- Rust 领域测试(最常用):`cargo test -p grading-core`
- 前端纯 reducer 单测:`npx vitest run`
- 类型检查:`npx vue-tsc --noEmit`
- 运行:`npm run tauri dev`

## 隐私

学生答卷图与成绩是隐私数据,一场考试的目录(`exam.db` + `images/`)**永不进版本管理**(见 `.gitignore`)。仓库里只有代码。
