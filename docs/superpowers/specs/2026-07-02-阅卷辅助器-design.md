# 阅卷辅助器 · 第一阶段设计文档

> 版本 v0.2 · 定稿（在 `plan.md` v0.1 基线上经一轮 grilling 修订）
> 日期：2026-07-02
> 定位：单人使用、本地优先、自带工具进场的桌面阅卷工作流工具。

本文档在 v0.1 基线上收紧了六处此前未钉死或内部有张力的设计分支，是后续实现计划（writing-plans）的依据。未在本文改动的部分，以 `plan.md` 为准。

---

## 0. 一句话（不变）

把奥赛纸质大题判卷里**所有非判断性的损耗**（翻页寻址、手眼往返录入、事后聚合）全部消灭，只留下"看题—给分"这一个纯判断动作，并让它在键盘上快到极致。价值锚点是**判分循环**，仅此。

---

## 1. 范围与边界（不变）

第一阶段唯一目标：**一个人，对着一批已扫好的干净图，把一场奥赛（<8 道大题）批到"极爽"，并一键出成绩表。**

非目标（推迟）与 v0.1 一致：成像/矫正、AI 判分、多用户/协作、二维码/条码/OCR 自动识别、页内切分/版面分析/OMR、dewarp。判据不变——动手前问："这是**我一个人批这场奥赛需要的**，还是**我想做的**？"

---

## 2. 技术选型（不变）

**Tauri v2 + Rust `grading-core` + SQLite + Vue3 + monolith 设计系统。** 复用 boothkernel 架构，零迁移成本。核心逻辑抽成独立 crate `grading-core`，经 Tauri IPC 暴露给前端；将来要多机，同一 core 外再包一层 Axum，不重写。默认单机、本地 SQLite、单写者，无并发问题。

---

## 3. 流水线（不变）：Setup → Ingest → Label → Grade → Export

v0.1 的五段流水线在 v0.2 显式补入了 **Exam Setup（考试设置）** 段（原 plan 隐含未写），并明确 Preprocess（成像）在第一阶段外包给扫描全能王：

```
① Setup    考试设置  —— 题目骨架 + 每题满分 + 快捷分数档位 + 花名册【新增段，判分前置】
② Ingest   导入      —— 读图片文件夹（最简，最后做）
③ Preprocess 成像    —— 【第一阶段跳过，外包给扫描全能王】
④ Label    打标签    —— 导花名册 + 姓名页定边界从名册选人 + 叠内页序=题号
⑤ Grade    判分      —— 横向批题键盘循环【命门，最先做】
⑥ Export   导出      —— CSV / PDF 成绩表 + 题目级得分率
```

## 4. 数据模型（**已修订**）

### 4.1 关键决策变更：v1 只存题目级总分，砍掉题内步级录入

v0.1 计划分步存 `score_item`（每个 rubric 步骤给了几分）。**v0.2 决定：第一版判分单元 `(学生, 题号)` 只存一个总分数字，不录题内步级得分。** 理由：奥赛判卷细化到"每道题"的诊断粒度已经足够；再细分到题内步骤，录入负担太重、且部分步骤本质连续给分拆不动。

- `score_item` / 未来的 `rubric_step`：**建表占位，v1 不录入、不依赖**，为将来的题内步级诊断留向前兼容的门。
- 判分不再逐步 toggle，而是靠**每题预置的"快捷分数档位"（score_preset）**一键给总分，配手动数字兜底（见 §5）。
- 因此题目级诊断（得分率）成立，题内步级诊断推迟到后续版本。

### 4.2 表结构

```sql
exam         (id, name, date)
problem      (id, exam_id, number, max_score, title)

-- 【新】每题的快捷分数档位，绑数字键槽 1..9
score_preset (id, problem_id, slot, label, points)
             -- slot ∈ 1..9；满分档(=max)、零分档(=0) 由系统自动预置

-- 学生来自导入的花名册（姓名 + 考号）；未绑定任何 page 者 = 缺考
student      (id, exam_id, name, exam_number, roster_order)

-- 一张扫描图一条；一个 (学生,题号) 可挂多张（溢出）
-- problem_number = 0 表示姓名页/封面（不判分，速览可见，做身份复核）
page         (id, exam_id, student_id, problem_number,
              image_path, seq, status)

-- 判分结果，判分单元 = (学生, 题号)，只存总分 + 状态
score        (id, student_id, problem_id, total, state,
              preset_id, grader, submitted_at)
             -- state ∈ {未判, 存疑, 已判}
             -- preset_id 记录用了哪个档位（手动输入时为 NULL）

-- 【占位·v1 不录入】未来题内步级诊断用
rubric_step  (id, problem_id, order, label, points)   -- 空表，前向兼容
score_item   (id, score_id, rubric_step_id, earned)   -- 空表，前向兼容
```

### 4.3 两个不变的要点

1. **判分单元是 `(学生, 题号)`，不是 page。** 大题溢出到第二面时，一个 `(学生,题号)` 挂多张 page，判分视图一起滚动显示。
2. **存储组织**：`/某场考试/{images/, exam.db}`，整个目录可拎走、归档、交接。导入时把图**拷进** `images/`，原件不动。

## 5. 命门：判分循环 UX（**已修订**）

> 唯一每天点几百上千次的地方，90% 的好用度在这里。动手前先拿纸把按键序列与界面框图画死。

### 5.1 模式：横向批题（不变）

选定一道题 → 系统把该题所有 `(学生,题号)` 单元排队 → 逐个呈现。连批同一题几十上百份，标准最稳、速度最快。

### 5.2 双模式给分（**新**）

- **爽批模式（默认）**：数字键 `1`–`9` = 当前题的**快捷分数档位**（Setup 里定义，如 `1=满分 / 2=前两问9 / 3=只建系4 / 4=零分`），一键落总分。
- **手动模式（兜底）**：按 `M`（或 `0`）进入手动输入，敲任意数字，覆盖档位、处理怪分。
- 档位是**每题各自的绝对数字列表**，不做跨题模板、不做"满分−n"相对表达式。总分实时显示在角上。

**`Enter` 上下文相关（必须钉死，别一键两义）**：焦点在**手动输入框内**时，`Enter` = 确认这个数字并**留在当前单元**（给你一眼复核的机会）；焦点**不在输入框**（爽批态）时，`Enter` = 提交并跳下一个学生（见 §5.3 轴1）。即手动给分要 `Enter` 两次：第一次确认数字、第二次前进。爽批档位一键即落分，`Enter` 直接前进。

### 5.3 两条导航轴与键位（**冲突已解决**）

判分存在两条独立导航轴，分开绑键：

| 键 | 动作 | 轴 |
|---|---|---|
| `1`–`9` | 选当前题的快捷档位 | 给分 |
| `M` / `0` | 进手动输入，敲数字回车 | 给分 |
| `Enter` / `空格` | 提交并跳**下一个学生**（同题） | 轴1 队列前进 |
| `Backspace` | 回**上一个学生**改分 | 轴1 队列后退 |
| `←` / `→` | **只读速览**当前学生的上/下一张答题纸 | 轴2 跨题翻看 |
| `↓` / `Esc` | 从速览跳回当前判分单元 | 轴2 复位 |
| `F` | 标记存疑待复看 | — |
| `J` | 跳到下一个存疑 | — |
| `G` | 打开**队列总览**，键入序号 / 点击任意单元直接跳过去 | 轴1 随机跳转 |
| 滚轮 / `+` `-` | 图像缩放 | — |

**关键语义**：轴2 速览**纯只读**——翻看该学生别的答题纸时，右侧评分面板始终锚定在当前 `(学生, 题号)`，数字键给的仍是这道题的分，绝不会误落到正瞄着的那页上。自动标签建好"题目-学生-页码"对应表做默认锚点与自动跳转；轴2 让你在锚点之外随时手翻兜底（处理"接着写到下一页"这类灵活情况）。

**队列随机跳转（`G`）**：`Enter/Backspace` 只能相邻前后、`J` 只能跳存疑——批到第 40 份想起第 5 份判错时，退 35 次不可接受。`G` 打开一个**队列总览**：列出本题所有单元及其状态（已判/存疑/未判），键入序号或点击即跳过去改。`J` 是"跳下一个存疑"的快捷子集。

**判分中发现 Label 错的回收路径**：轴2 是只读的，若速览时发现某页题号/学生绑错，**当场改不了**——记下后回 **Label 阶段**修正（Label 是独立阶段，改完重进判分）。不在判分只读界面里塞标签编辑，避免两个阶段职责混淆。

### 5.4 单屏布局（不变，见 v0.1 §5.2）

左侧答卷图像（滚轮缩放、溢出页下方续接），右侧当前题的档位列表 + 实时总分，底部进度条 + 存疑操作。

## 7. 打标签方案 Label（**已修订**）

v0.1 默认"扫描序 = 名单序"。**v0.2 修正：交卷是人肉乱序的，纯顺序锚不住"这叠是谁"。** 拆开 v0.1 §7 里被捆在一起的两个假设：

1. **这叠是谁（跨学生身份）**——乱序，纯顺序不成立，靠**姓名页 + 花名册**绑定。
2. **叠内这页是第几题（学生内部）**——只要每个学生的卷子整叠码在一起（学生间乱序、学生内不散），`第 k 页 = 第 k 题` 仍成立。

**利用一个好习惯**：每个学生答题卡第一页是**姓名页**。它让学生边界从"数张数猜"变成"肉眼可见"（看到姓名页 = 新学生开始），Label 因此大幅简化。

### 7.0 承重不变量：叠内答题页与题号严格一一对应（**必须显式声明**）

叠内自动编题号是纯位置自增，其成立的**唯一前提**是：`每个学生的答题页数 == 题数 N`，题目密集无缺口。奥赛学生放弃某题是常态，若 `[姓名页, Q1, Q2, Q4, Q5]`（跳了 Q3 无占位），纯自增会把 Q4 静默标成"题3"、其后全体后移——**静默错位**。故用**物理 + 软件叠加**守住这条不变量：

- **物理层（纪律，保效率 + 坐实语义）**：**缺答塞空白页占位**。每个学生严格 = 姓名页 + N 张答题页（缺答那题放一张空白页）。空白页照常进判分队列被打 0（Setup 里配一个"空白=0"档位），于是"**没答=真 0 分**"与"**没批=未判**"从物理上就分开了（正好接 §9 三态）。
- **软件层（兜底，保正确，不可退让）**：Label 绑定学生时工具已知本场 N 题，**绝不盲目自增**——`答题页数 == N` 才走自增快路径；`≠ N`（漏塞空白页 / 多扫 / 少扫）立即**报警并强制你逐页指认题号 / 标出缺哪题**，绝不默默编号。
- **键位对称（修掉 §5 之前只能"退"不能"进"的不对称）**：`C` = 接上一题（溢出，计数器**不前进**）；`N` = 缺题/跳题（声明"这页是题 k、题 k−1 空缺"，计数器**跳进**）。两个方向都给。

> 正确性押在软件层（对不上就报警），不押在"人一定记得塞空白页"这条线下纪律上；塞空白页只是让常态一键化、并顺带区分没答/没批。

### 7.1 修订后的 Label 流程

- 花名册已在 **Exam Setup 段导入**（见 §8），是 §4 `student` 表来源，也是**到场清单**（见 §7.3）。Label 段直接用它，不再单独导入。
- 卷子**按学生成叠**扫（学生间顺序随意），叠内首页姓名页、其后按题号序、缺答塞空白页（§7.0）。
- **Label 界面 = 单图浏览屏**，`←`/`→` 按扫描序（文件名自然排序）逐张翻，与判分速览同一套肌肉记忆：
  - 翻到**姓名页**（新学生起点）按 `S`/`Enter` → 标记"新学生从这里开始"，弹出**花名册选人**（搜姓名 / 键几位考号定位，回车确认绑定）。姓名页本身记为 `problem_number = 0`，不判分，留存挂该学生名下、速览可见做身份复核。
  - 两个姓名页之间的答题页，在 `答题页数 == N` 时**自动按序编题号** `第1题…第N题`；对不上则按 §7.0 强制指认。
  - 溢出用 `C`（接上一题、不进），跳题用 `N`（跳进），见 §7.0。
- **必须有的界面**：一个"叠 ↔ 学生 ↔ 各页题号"的可视化确认总表，翻完扫一眼、拖一拖纠错位；页数对不上的叠**高亮报警**。

### 7.2 身份绑定方式：从花名册选人（不是手填考号）

姓名页上有可读的名字，故绑定走**从花名册选人**（搜姓名或键考号定位），比对着卷面找考号手敲更顺、更不易错。花名册是身份的单一真源。

**花名册对不上的逃生口**：姓名页上是名册里查无此人的名字（转学生、临时加考、名册漏录、名字打错）时，允许**临时新增学生**（补一条 `student` 记录）或**标记"待匹配"**。待匹配的叠在确认总表和导出前单独列出，逼你收尾前处理，不让它静默漏掉。

### 7.3 缺考检测（花名册的副产品）

花名册同时是到场清单：**名单里有、但 Label 结束仍无任何 page 绑定的学生 = 缺考/没交卷。** 工具在 Label 确认界面和导出前主动列出这些人，与"已交但未判"区分开（见 §9.1）。

### 7.4 推迟项

- **OCR 姓名页 / 手写考号**：v1 不做（手写识别不可靠，贴着非目标线），身份绑定一律**人工从花名册选人**。将来做也只当辅助校验。
- **二维码/条码识别**：中期优化，第一阶段不做。永不依赖学生手写的页码/题号。

## 8. 考试设置 Exam Setup（**新增段**）

Exam Setup 是"这场考试是什么"的完整定义——题目、满分、档位、**花名册**同属考试骨架，全在这一层录好。Setup 在导入/判分前做一次：

- **录题目**：题号、标题、**满分**。
- **配快捷档位**：每题一组 `(标签, 绝对分值)` 绑到数字键槽 `1`–`9`；`满分`(=max)、`零分`(=0)、`空白`(=0，供 §7.0 空白占位页用) 自动预置，其余手敲（如第3题 `2=前两问9 / 3=只建系4`）。档位是每题各自的绝对数字，砍掉步级结构后"前两问=9"对软件就是数字 `9`。
- **导花名册**：`姓名, 考号` CSV → 写入 §4 的 `student` 表。花名册既是 Label 段绑身份的选人来源，也是**到场清单**（缺考检测，见 §7.3）。Label 段不再单独导名册，直接用 Setup 里这份。

**档位设计指引（找回一部分被 §4.1 让掉的步级诊断，零成本）**：`score` 表存了 `preset_id`（§4.2），所以档位本身携带诊断信号——若第3题档位 `3=只建系4`，一句 `WHERE preset_id = 该档位` 就能捞出"卡在建系那步的人"。据此两条：

1. **按"卡在哪一步"设计档位，而非只按分值凑数字**——让 `preset_id` 的诊断价值最大化（"只建系""建系+列方程""全对"比"4分档""9分档"有用得多）。
2. **手动输入 `preset_id = NULL`，这个信号在手动模式下丢失**——所以**能用档位就别手动**，把手动留给真正的怪分。

## 9. 导出 Export（**已修订**）

### 9.1 判分单元三态与"绝不静默填 0"

判分单元有三态：`已判`（有分）/ `存疑`（可有临时分，标记待复看）/ `未判`（从没碰过）。此外学生层面还有一个 `缺考` 态（花名册里有、但无任何 page 绑定，见 §7.3），与"已交但未判"严格区分。

- **导出时 `未判`/`存疑` 绝不静默填 0**——否则一个被跳过没批的学生会在成绩表里变成"这题真得 0 分"的惨案且查不出。
- 导出前先算**覆盖率 + 到场情况**并弹确认，例如：`花名册 50 人，缺考 1；已交 49 人 × 6 题 = 294 单元，已判 291、存疑 2、未判 1，仍要导出吗？`
- **软提示不硬拦**（单人工具，你为结果负责）；导出表里 `未判/存疑` 格子**留空或标 `?`**、`缺考` 学生整行标 `缺考`，让异常一眼可见。

**总分与排名在带洞时的语义（必须钉死，别给出看着像真的错排名）**：

- **临时分计入临时总分**：`存疑` 单元你敲的临时分是真数，计入该生总分显示，但单元带存疑标；`未判` 单元不贡献分（留空，不当 0）。故有洞时总分是**下界/临时值**，不是定值。
- **排名只在"该生所有单元均已判"时对该生计算**：只要某生还有 `未判/存疑`，其总分未定，**排名对他无意义**。因此——**未达全部已判前不出排名列**（只出各题分与临时总分，排名整列留空并标"不完整：先批完再排"）；全已判后排名才计算。绝不用带洞数据排出一个像真的名次。

### 9.2 导出内容

- **CSV**：列 = `姓名, 考号, 题1, 题2, …, 题N, 总分, [排名]`；`未判/存疑` 单元留空或标 `?`；`排名` 列仅在全部已判时出现，否则整列留空标"不完整"（见 §9.1）。附**题目级汇总**：每题 `平均分 / 满分 / 得分率`。
- **PDF**：上述两张表的可打印版（成绩排名表 + 各题得分率），monolith 冷淡风、高信息密度，v1 不做花哨排版。

### 9.3 Ingest 细节

读图片文件夹按**文件名自然排序**（`1,2,…,10,11` 而非 `1,10,11,2`）定扫描序；支持 `jpg/png/webp`；导入时把图拷进该场考试的 `images/`。

---

## 10. AI 层（不变）

第一阶段不做。将来做也守死一条线：**AI 当分诊台，不当法官，最终分永远是你按的键。** 架构硬约束（写死在 core 里）：AI 模块只能写建议字段和排序，**永远不能写 `score.total`**。仅允许"组织（聚相似答卷连批）"与"质检（挑很像但给分差很多的对子提醒复看）"两件安全的事。

## 11. 构建顺序与里程碑（**微调**）

先验证最核心的爽感，再补齐管道。因 §5 判分改用档位，M1 需要一个**极简 Setup**（能录题目+档位）作前置。

| 里程碑 | 内容 | 验收 |
|---|---|---|
| **M0 极简 Setup** | 能录题目、满分、每题快捷档位、导花名册（够 M1/M2 用即可） | 建出一场假考试的骨架、档位与学生名单 |
| **M1 判分循环** | 假数据须够"厚"：不只是一道题的队列，还要含**至少一个完整学生的跨题 page 集**（多道题各有图），否则轴2 只读速览（翻看该生别的答题纸）无法验收。含 `M`/`Enter` 上下文语义、`G` 队列跳转 | 全键盘横向批完一道题、档位/手动给分、`Enter` 上下文正确、实时总分、回改、存疑、`G` 随机跳转、轴2 跨题速览 |
| **M2 打标签** | §7 姓名页定边界从名册（M0 已导）选人 + 叠内页序=题号 + 溢出续页 + 确认总表 + 缺考检测 | 导入乱序成叠图能正确绑人切题，错位/溢出一眼可修，缺考自动列出 |
| **M3 导出** | §9 CSV/PDF + 题目级得分率 + 三态覆盖率确认 | 一键出总分排名表 + 各题得分率，异常留空不填0 |
| **M4 导入** | 读图片文件夹（最简，最后补） | 指向扫描全能王导出目录即可开批 |

> M1 先行的理由：判分循环是价值锚点，也最能暴露 UX 问题。

## 12. 风险与对策（**更新**）

| 风险 | 对策 |
|---|---|
| **过度设计**（最大风险） | §1 non-goal 硬边界；每个新功能问"需要 vs 想做" |
| **交卷乱序导致错人** | §7 姓名页定边界 + 从花名册选人；确认总表可视化纠错；名册对不上有临时新增/待匹配逃生口 |
| **有人没交卷被漏掉** | §7.3 花名册当到场清单，Label 后自动列出缺考者，与"未判"区分 |
| **跳题致自动编号静默错位**（承重） | §7.0 不变量：物理缺答塞空白页 + 软件页数≠N 强制指认、绝不盲目自增；键位 `C`退/`N`进对称 |
| **溢出页遗漏** | 数据模型 `(学生,题号)` 挂多 page；`C` 接上一题 + 确认界面手动合并 |
| **静默填 0 惨案** | §9 三态 + `缺考` 态；导出前覆盖率确认；异常留空不填 0；带洞不出排名 |
| **`Enter` 一键两义** | §5.2 上下文相关：输入框内=确认留原地，爽批态=提交前进 |
| **并发妖怪的应激反应** | 单机单用户本地 SQLite，单写者天然成立，怎么简单怎么来 |
| **交付物带第三方水印** | 到中期再把成像段换成自有 jscanify/OpenCV 链或 ADF 扫描仪 |

## 13. 第一阶段完成的定义（DoD）

指向一个装着扫描全能王导出图的文件夹 + 一份花名册 CSV，能够：

1. 在 Setup 里建好题目骨架、满分、每题快捷档位与花名册；
2. 导入乱序成叠的图，靠姓名页定边界从花名册选人、靠叠内页序切题，页数≠N 报警强制指认（缺答空白页占位），溢出/跳题/错位可一眼手动修，缺考者自动列出；
3. 全键盘、横向批题、档位/手动双模式给总分（`Enter` 上下文语义正确），两轴导航 + `G` 随机跳转，一个人批得极爽；
4. 一键导出总分表 + 各题得分率，三态清晰、异常留空不填 0、带洞不出排名；
5. 整场数据是一个可拎走的自包含文件夹。

达成即第一阶段结束。题内步级诊断（score_item）、OCR/二维码识别、AI 分诊、自有成像链、多用户，都是**验证了第一阶段爽感之后**才谈的下一阶段。

---

*把不确定性挡在你能控制的物理边界之外（姓名页定边界、花名册当到场清单、三态可见），而不是在下游用算法去救——这是贯穿整套设计的唯一原则。*
