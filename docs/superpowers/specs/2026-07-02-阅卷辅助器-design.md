# 阅卷辅助器 · 第一阶段设计文档

> 版本 v0.2 · 定稿（在 `plan.md` v0.1 基线上经一轮 grilling 修订）
> 日期：2026-07-02
> 定位：单人使用、本地优先、自带工具进场的桌面阅卷工作流工具。

本文档在 v0.1 基线上收紧了六处此前未钉死或内部有张力的设计分支，是后续实现计划（writing-plans）的依据。未在本文改动的部分，以 `plan.md` 为准。

---

## 0. 一句话（不变）

把奥赛纸质大题判卷里**所有非判断性的损耗**（翻页寻址、手眼往返录入、事后聚合）全部消灭，只留下"看题—给分"这一个纯判断动作，并让它在键盘上快到极致。价值锚点是**判分循环**，仅此。

---

## 1. 范围与边界（不变）

第一阶段唯一目标：**一个人，对着一批已扫好的干净图，把一场奥赛（<8 道大题）批到"极爽"，并一键出成绩表。**

非目标（推迟）与 v0.1 一致：成像/矫正、AI 判分、多用户/协作、二维码/条码/OCR 自动识别、页内切分/版面分析/OMR、dewarp。判据不变——动手前问："这是**我一个人批这场奥赛需要的**，还是**我想做的**？"

---

## 2. 技术选型（不变）

**Tauri v2 + Rust `grading-core` + SQLite + Vue3 + monolith 设计系统。** 复用 boothkernel 架构，零迁移成本。核心逻辑抽成独立 crate `grading-core`，经 Tauri IPC 暴露给前端；将来要多机，同一 core 外再包一层 Axum，不重写。默认单机、本地 SQLite、单写者，无并发问题。

---

## 3. 流水线（不变）：Setup → Ingest → Label → Grade → Export

v0.1 的五段流水线在 v0.2 显式补入了 **Exam Setup（考试设置）** 段（原 plan 隐含未写），并明确 Preprocess（成像）在第一阶段外包给扫描全能王：

```
① Setup    考试设置  —— 建题目骨架 + 每题满分 + 快捷分数档位【新增段，判分前置】
② Ingest   导入      —— 读图片文件夹（最简，最后做）
③ Preprocess 成像    —— 【第一阶段跳过，外包给扫描全能王】
④ Label    打标签    —— 导花名册 + 逐叠手填考号定边界与身份 + 叠内页序=题号
⑤ Grade    判分      —— 横向批题键盘循环【命门，最先做】
⑥ Export   导出      —— CSV / PDF 成绩表 + 题目级得分率
```

## 4. 数据模型（**已修订**）

### 4.1 关键决策变更：v1 只存题目级总分，砍掉题内步级录入

v0.1 计划分步存 `score_item`（每个 rubric 步骤给了几分）。**v0.2 决定：第一版判分单元 `(学生, 题号)` 只存一个总分数字，不录题内步级得分。** 理由：奥赛判卷细化到"每道题"的诊断粒度已经足够；再细分到题内步骤，录入负担太重、且部分步骤本质连续给分拆不动。

- `score_item` / 未来的 `rubric_step`：**建表占位，v1 不录入、不依赖**，为将来的题内步级诊断留向前兼容的门。
- 判分不再逐步 toggle，而是靠**每题预置的"快捷分数档位"（score_preset）**一键给总分，配手动数字兜底（见 §5）。
- 因此题目级诊断（得分率）成立，题内步级诊断推迟到后续版本。

### 4.2 表结构

```sql
exam         (id, name, date)
problem      (id, exam_id, number, max_score, title)

-- 【新】每题的快捷分数档位，绑数字键槽 1..9
score_preset (id, problem_id, slot, label, points)
             -- slot ∈ 1..9；满分档(=max)、零分档(=0) 由系统自动预置

-- 学生来自导入的花名册（姓名 + 考号）
student      (id, exam_id, name, exam_number, roster_order)

-- 一张扫描图一条；一个 (学生,题号) 可挂多张（溢出）
page         (id, exam_id, student_id, problem_number,
              image_path, seq, status)

-- 判分结果，判分单元 = (学生, 题号)，只存总分 + 状态
score        (id, student_id, problem_id, total, state,
              preset_id, grader, submitted_at)
             -- state ∈ {未判, 存疑, 已判}
             -- preset_id 记录用了哪个档位（手动输入时为 NULL）

-- 【占位·v1 不录入】未来题内步级诊断用
rubric_step  (id, problem_id, order, label, points)   -- 空表，前向兼容
score_item   (id, score_id, rubric_step_id, earned)   -- 空表，前向兼容
```

### 4.3 两个不变的要点

1. **判分单元是 `(学生, 题号)`，不是 page。** 大题溢出到第二面时，一个 `(学生,题号)` 挂多张 page，判分视图一起滚动显示。
2. **存储组织**：`/某场考试/{images/, exam.db}`，整个目录可拎走、归档、交接。导入时把图**拷进** `images/`，原件不动。

## 5. 命门：判分循环 UX（**已修订**）

> 唯一每天点几百上千次的地方，90% 的好用度在这里。动手前先拿纸把按键序列与界面框图画死。

### 5.1 模式：横向批题（不变）

选定一道题 → 系统把该题所有 `(学生,题号)` 单元排队 → 逐个呈现。连批同一题几十上百份，标准最稳、速度最快。

### 5.2 双模式给分（**新**）

- **爽批模式（默认）**：数字键 `1`–`9` = 当前题的**快捷分数档位**（Setup 里定义，如 `1=满分 / 2=前两问9 / 3=只建系4 / 4=零分`），一键落总分。
- **手动模式（兜底）**：按 `M`（或 `0`）进入手动输入，敲任意数字回车，覆盖档位、处理怪分。
- 档位是**每题各自的绝对数字列表**，不做跨题模板、不做"满分−n"相对表达式。总分实时显示在角上。

### 5.3 两条导航轴与键位（**冲突已解决**）

判分存在两条独立导航轴，分开绑键：

| 键 | 动作 | 轴 |
|---|---|---|
| `1`–`9` | 选当前题的快捷档位 | 给分 |
| `M` / `0` | 进手动输入，敲数字回车 | 给分 |
| `Enter` / `空格` | 提交并跳**下一个学生**（同题） | 轴1 队列前进 |
| `Backspace` | 回**上一个学生**改分 | 轴1 队列后退 |
| `←` / `→` | **只读速览**当前学生的上/下一张答题纸 | 轴2 跨题翻看 |
| `↓` / `Esc` | 从速览跳回当前判分单元 | 轴2 复位 |
| `F` | 标记存疑待复看 | — |
| `J` | 跳到下一个存疑 | — |
| 滚轮 / `+` `-` | 图像缩放 | — |

**关键语义**：轴2 速览**纯只读**——翻看该学生别的答题纸时，右侧评分面板始终锚定在当前 `(学生, 题号)`，数字键给的仍是这道题的分，绝不会误落到正瞄着的那页上。自动标签建好"题目-学生-页码"对应表做默认锚点与自动跳转；轴2 让你在锚点之外随时手翻兜底（处理"接着写到下一页"这类灵活情况）。

### 5.4 单屏布局（不变，见 v0.1 §5.2）

左侧答卷图像（滚轮缩放、溢出页下方续接），右侧当前题的档位列表 + 实时总分，底部进度条 + 存疑操作。

## 6. 采集方案（不变）

第一阶段外包给扫描全能王/夸克扫描王免费版：单页/单学生出图，不用多页 PDF 功能（绕开页数限制），忍受角落水印（不碰作答区）。不下破解版。手机只负责把图弄干净，不做命名/分组。中期演进再把成像段收进本工具（jscanify / OpenCV）。

## 7. 打标签方案 Label（**已修订**）

v0.1 默认"扫描序 = 名单序"。**v0.2 修正：交卷是人肉乱序的，纯顺序锚不住"这叠是谁"。** 拆开 v0.1 §7 里被捆在一起的两个假设：

1. **这叠是谁（跨学生身份）**——乱序，纯顺序不成立，靠**读考号绑花名册**。
2. **叠内这页是第几题（学生内部）**——只要每个学生的卷子整叠码在一起（学生间乱序、学生内不散），`第 k 页 = 第 k 题` 仍成立。

### 7.1 修订后的 Label 流程

- 批前导入**花名册 CSV**（`姓名, 考号`）——即 §4 的 `student` 表来源。
- 卷子**按学生成叠**扫（学生间顺序随意），叠内按题号序。
- Label 时**逐叠绑定**：在确认界面**手填这叠的考号** → 匹配花名册里的人；敲考号这个动作**同时定义叠的边界与身份**（不再假设"每叠固定 N 张"自动切——乱序前提下该假设太脆）。
- 叠内 `第 k 页 = 第 k 题`；溢出（一题写到第二面）在确认界面**手动把相邻两张合并**进同一个 `(学生,题号)` 单元。
- **必须有的界面**：一个"叠 ↔ 学生 ↔ 各页题号"的可视化确认页，导入后扫一眼、拖一拖纠错位。

### 7.2 推迟项

- **OCR 手写考号**：v1 不做（手写数字识别不可靠，贴着非目标线），身份绑定一律**手填**。将来做也只当辅助校验。
- **二维码/条码识别**：中期优化，第一阶段不做。永不依赖学生手写的页码/题号。

## 8. 考试设置 Exam Setup（**新增段**）

判分靠档位一键给分，档位得先有地方定义。Setup 在导入/判分前做一次：

- **录题目**：题号、标题、**满分**。
- **配快捷档位**：每题一组 `(标签, 绝对分值)` 绑到数字键槽 `1`–`9`；`满分`(=max)、`零分`(=0) 自动预置，其余手敲（如第3题 `2=前两问9 / 3=只建系4`）。
- 档位是每题各自的绝对数字，砍掉步级结构后"前两问=9"对软件就是数字 `9`。

## 9. 导出 Export（**已修订**）

### 9.1 判分单元三态与"绝不静默填 0"

判分单元有三态：`已判`（有分）/ `存疑`（可有临时分，标记待复看）/ `未判`（从没碰过）。

- **导出时 `未判`/`存疑` 绝不静默填 0**——否则一个被跳过没批的学生会在成绩表里变成"这题真得 0 分"的惨案且查不出。
- 导出前先算**覆盖率**并弹确认，例如：`50 人 × 6 题 = 300 单元，已判 297、存疑 2、未判 1，仍要导出吗？`
- **软提示不硬拦**（单人工具，你为结果负责）；导出表里这些格子**留空或标 `?`**，让异常一眼可见。

### 9.2 导出内容

- **CSV**：列 = `姓名, 考号, 题1, 题2, …, 题N, 总分, 排名`；`未判/存疑` 单元留空或标 `?`。附**题目级汇总**：每题 `平均分 / 满分 / 得分率`。
- **PDF**：上述两张表的可打印版（成绩排名表 + 各题得分率），monolith 冷淡风、高信息密度，v1 不做花哨排版。

### 9.3 Ingest 细节

读图片文件夹按**文件名自然排序**（`1,2,…,10,11` 而非 `1,10,11,2`）定扫描序；支持 `jpg/png/webp`；导入时把图拷进该场考试的 `images/`。

## 10. AI 层（不变）

第一阶段不做。将来做也守死一条线：**AI 当分诊台，不当法官，最终分永远是你按的键。** 架构硬约束（写死在 core 里）：AI 模块只能写建议字段和排序，**永远不能写 `score.total`**。仅允许"组织（聚相似答卷连批）"与"质检（挑很像但给分差很多的对子提醒复看）"两件安全的事。

## 11. 构建顺序与里程碑（**微调**）

先验证最核心的爽感，再补齐管道。因 §5 判分改用档位，M1 需要一个**极简 Setup**（能录题目+档位）作前置。

| 里程碑 | 内容 | 验收 |
|---|---|---|
| **M0 极简 Setup** | 能录题目、满分、每题快捷档位（够 M1 用即可） | 建出一场假考试的骨架与档位 |
| **M1 判分循环** | 用假数据（几十张图 + M0 的档位），把 §5 双模式 + 两轴键位做到"一个人批起来极爽" | 全键盘横向批完一道题、档位/手动给分、实时总分、回改、存疑、只读速览 |
| **M2 打标签** | §7 花名册导入 + 逐叠手填考号定边界身份 + 叠内页序=题号 + 溢出合并 + 确认界面 | 导入乱序成叠图能正确绑人切题，错位/溢出一眼可修 |
| **M3 导出** | §9 CSV/PDF + 题目级得分率 + 三态覆盖率确认 | 一键出总分排名表 + 各题得分率，异常留空不填0 |
| **M4 导入** | 读图片文件夹（最简，最后补） | 指向扫描全能王导出目录即可开批 |

> M1 先行的理由：判分循环是价值锚点，也最能暴露 UX 问题。

## 12. 风险与对策（**更新**）

| 风险 | 对策 |
|---|---|
| **过度设计**（最大风险） | §1 non-goal 硬边界；每个新功能问"需要 vs 想做" |
| **交卷乱序导致错人** | §7 手填考号即定边界+身份；确认界面全表可视化纠错 |
| **溢出页遗漏** | 数据模型 `(学生,题号)` 挂多 page；确认界面手动合并 |
| **静默填 0 惨案** | §9 三态模型 + 导出前覆盖率确认 + 异常留空不填 0 |
| **并发妖怪的应激反应** | 单机单用户本地 SQLite，单写者天然成立，怎么简单怎么来 |
| **交付物带第三方水印** | 到中期再把成像段换成自有 jscanify/OpenCV 链或 ADF 扫描仪 |

## 13. 第一阶段完成的定义（DoD）

指向一个装着扫描全能王导出图的文件夹 + 一份花名册 CSV，能够：

1. 在 Setup 里建好题目骨架、满分与每题快捷档位；
2. 导入乱序成叠的图，靠手填考号正确绑人、靠叠内页序切题，溢出/错位可一眼手动修；
3. 全键盘、横向批题、档位/手动双模式给总分，两轴导航，一个人批得极爽；
4. 一键导出总分排名表 + 各题得分率，三态清晰、异常留空不填 0；
5. 整场数据是一个可拎走的自包含文件夹。

达成即第一阶段结束。题内步级诊断（score_item）、OCR/二维码识别、AI 分诊、自有成像链、多用户，都是**验证了第一阶段爽感之后**才谈的下一阶段。

---

*把不确定性挡在你能控制的物理边界之外（手填考号、每叠人肉成叠、三态可见），而不是在下游用算法去救——这是贯穿整套设计的唯一原则。*
