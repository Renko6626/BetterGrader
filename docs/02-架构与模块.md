# 02 · 架构与模块

面向要改代码的开发者。先看三层架构和数据流，再是数据模型与不变量，然后是模块 / 命令 / 前端的逐一参考。

## 三层解耦架构

```
┌─────────────────────────────────────────────┐
│  src/ ── Vue3 + TS + naive-ui 前端            │  视图 + 纯键盘 reducer + api.ts(typed invoke)
├─────────────────────────────────────────────┤
│  src-tauri/src/commands.rs ── 薄 IPC 层        │  锁状态、转发、文件 IO；无业务逻辑
├─────────────────────────────────────────────┤
│  crates/grading-core/ ── 纯领域逻辑（单一真源） │  不知道 Tauri/前端存在，独立编译 + 独立测试
└─────────────────────────────────────────────┘
```

三层**刻意解耦**：

1. **`crates/grading-core/`** —— 纯领域逻辑，单一真源。不依赖 Tauri/前端，`cargo test -p grading-core` 独立跑（几乎所有逻辑的测试都在这里）。**文件系统/IO 不进这一层**，留在命令层。
2. **`src-tauri/src/commands.rs`** —— 薄 IPC 垫片。每个 `#[tauri::command]` 锁共享状态、委托给 `grading-core`，自己不做业务判断。
3. **`src/`** —— Vue3 前端。四个视图（Setup/Label/Grade/Export）；`api.ts` 是 typed `invoke` 包装；两个**纯键盘 reducer** 是判分/标注的命门。

## 五段流水线（§3）

```
设置 Setup ──► 摄入 Ingest / PDF 导入 ──► 标注 Label ──► 判分 Grade ──► 导出 Export
(题目+满分+           (产出 page 行)         (page 绑到           (键盘横向          (CSV 真文件 +
 档位+评分标准                              (学生,题号))         批题)             可打印报表)
 +花名册)
```

两条导入路径**收敛到同一「已标注 page」形态**，下游完全不区分来源：
- **图片文件夹 Ingest**：创建**未标注** page（`status='ingested'`，学生/题号留空），由「标注」人工填。
- **PDF 导入**：每个 PDF = 一个学生（文件名去扩展名 = 姓名），`page_index → problem_number`（第 0 页 = 姓名页），渲染成 JPEG 后**直接建已标注 page**（`status='labeled'`）。

## 数据模型（SQLite）

`grading-core/src/db.rs` 打开连接时执行 `PRAGMA foreign_keys=ON` + 整段 `schema.sql`，再跑幂等 `migrate`。**没有真正的外键 REFERENCES**，完整性靠 `UNIQUE` 约束 + 命名约定维持。共 8 张表：

| 表 | 关键列 | 说明 |
|---|---|---|
| **exam** | `id`, `name NOT NULL`, `date?` | 一场考试。**单库单场**（`ensure_exam` 保证只有一行）。 |
| **problem** | `exam_id`, `number`, `title?`, `max_score`, `rubric?`, `UNIQUE(exam_id,number)` | 题目。`rubric`（Markdown 评分标准）由 migrate 补列。 |
| **score_preset** | `problem_id`, `slot`, `label`, `points`, `UNIQUE(problem_id,slot)` | 每题快捷档位，绑数字键 1..9。 |
| **student** | `exam_id`, `name NOT NULL`, `exam_number?`, `roster_order?` | 来自花名册。无 page 绑定者 = 缺考。 |
| **page** | `exam_id`, `student_id?`, `problem_number?`, `image_path NOT NULL`, `seq?`, `status?` | 一张扫描图一行。`problem_number=0`=姓名页。`status`∈`ingested`/`labeled`。`student_id`/`problem_number` **可空**（未标注中间态）。 |
| **score** | `student_id`, `problem_id`, `total?`, `state NOT NULL DEFAULT 'Ungraded'`, `preset_id?`, `comment?`, `grader?`, `submitted_at?`, `UNIQUE(student_id,problem_id)` | 判分单元 =（学生,题号），只存**总分 + 状态**。手动给分时 `preset_id=NULL`。`comment` 由 migrate 补列。`grader` 已定义但当前无写入路径。 |
| **rubric_step** | `problem_id`, `order`, `label`, `points`（全可空） | **占位·v1 不用**：未来题内步级诊断。 |
| **score_item** | `score_id`, `rubric_step_id`, `earned?` | **占位·v1 不用**：与 rubric_step 配套。 |

### 跨文件不变量（改代码前务必记住）

1. **一场考试 = 一个自包含目录** `{exam.db, images/}`（可整体拎走归档）。
2. **`page.image_path` 只存文件名**，读取时对 `<dir>/images/` 解析（`read_image`）。配合文件名护栏，图片永不逃出 `images/`。
3. **判分单元 = `(student, problem)`**（`score` 表 `UNIQUE(student_id,problem_id)`）。
4. **`score.total` 只由 `set_score` 写**——评语/导入/「AI」都不写它。`set_comment` 只碰 `comment`。
5. **三态 `Ungraded`/`Flagged`/`Graded` + 学生层 `Absent`**。**`Absent` 不是存储态**，是导出时派生（学生无任何 page 即缺考，`export.rs` 填 `"Absent"`）。导出还强制「`Ungraded` 单元 `total` 恒为 `None`」，即便库里残留脏 total 也在导出时清成 None。
6. **`page.student_id` / `page.problem_number` 可空**（未标注）。
7. **`problem_number = 0` = 姓名页/封面**，判分队列与答题页计数处处排除。`shift_student_problems` 平移时只动 `problem_number IS NOT NULL` 的页。
8. **两条导入路径产出同构 page 行**，下游 build_queue/判分/导出/§7.0 计数一律不分来源。

## `grading-core` 模块参考

`lib.rs` 导出：`db, models, setup, grading, fake, persist, export, ingest, label`，并 `pub use db::Db;` + `pub use models::*;`。除 `models.rs` 外每个模块都有单元测试。

### db.rs — 连接与幂等迁移
- `Db::open(path)` / `Db::open_in_memory()` → 建表 + `migrate`。
- `migrate(conn)`：对既有库幂等补列（`score.comment`、`problem.rubric`），内部 `add_column_if_missing` 用 `pragma_table_info` 检测存在才 `ALTER`，可重复调用。

### models.rs — 领域类型
serde 可序列化结构 + `enum ScoreState { Ungraded, Flagged, Graded }`（`as_str`/`from_str`，未知串兜底 `Ungraded`）。核心：`Problem{…,rubric:Option<String>}`、`GradingUnit{ student_id, student_name, problem_id, problem_number, pages:Vec<String>, total:Option<i64>, state, preset_id, comment }`、导出用 `ExportData/StudentRow/Cell/ProblemStat/Coverage`、标注用 `PageRow/StackRow/LabelSummary`。

### setup.rs — 考试骨架
- `create_exam` / `list_problems`（带 `rubric`）/ `list_presets` / `import_roster`（单事务，`roster_order`=输入下标）/ `list_students`。
- `add_problem(...)`：**单事务**内插题目 + **自动预置三档**（键 7=满分=max、键 8=零分、键 9=空白）；全成功或全回滚。
- `add_preset` / `set_problem_rubric`（`trim()` 为空即存 NULL = 清空）。
- ⚠️ **改题目满分的 `set_problem_max` 不在这里**——它在 `commands.rs` 里用裸 SQL 实现（UPDATE `problem.max_score` 时**同步**更新 label='满分' 档位的 points）。

### grading.rs — 判分队列与三态
- `build_queue(db, exam_id, problem_number) → Vec<GradingUnit>`：学生按 `roster_order`；每 (学生,题号) 的 page 按 `seq` 汇总；无 score 行则默认 `(total=None, Ungraded, preset=None, comment=None)`——**队列绝不默认 0**。
- `set_score(...)`：UPSERT（`ON CONFLICT(student_id,problem_id)`）写 total/state/preset_id + `submitted_at`，**不碰 comment**。
- `set_comment(...)`：UPSERT，插入时 state 兜底 `Ungraded`，冲突时**只更新 comment**，不冲掉分数。
- `student_pages(...)`：该生所有页按 seq（轴 2 速览用）。

### export.rs — 导出构建与 CSV
- `build_export(db, exam_id) → ExportData`：缺考 = page 数为 0；只有 `Graded/Flagged` 且 `total=Some` 才计入总分与题目均分累加器（null-total 的 Flagged 不拖低均分）；**排名规则** = 存在非缺考学生且**全部非缺考学生 complete**（每题都 Graded），标准竞赛名次（1,1,3）；`ProblemStat` 在 `cnt>0` 才算 avg/rate，否则 None（不除零）。
- `export_to_csv(data, include_comments) → String`：表头 `姓名,考号,题n…,总分`，可判排名时追加 `排名`；开评语时追加 `题n评语`——**所有分列在前、所有评语列在后**。单元渲染：缺考整行「缺考」；Graded→数字；Flagged→`{t}?`（null→`?`）；Ungraded→空。（CSV 写文件时命令层前置 UTF-8 BOM。）

### persist.rs — 单库单场
- `ensure_exam`（已有则返回 MIN id，不新建）/ `exam_count` / `exam_info`。

### ingest.rs — 自然排序与摄入
- `sort_scan_order(files)`：用 `compare_nat` 自然排序（数字按值而非字典序，`"2" < "10"`，数字开头排字母前）。
- `next_seq`（`MAX(seq)+1`，空表 0）/ `add_ingested_page`（插 `student_id=NULL, problem_number=NULL, status='ingested'`）。

### label.rs — 标注与承重计数
- `list_pages` / `set_page_label`（给了 student_id → `labeled`，否则退回 `ingested`）/ `add_student`（`roster_order=MAX+1`）/ `add_labeled_page`（PDF 导入直接落已标注行）。
- `labeling_summary(...) → LabelSummary`（**§7.0 承重计数**）：`answer_pages` = 该生 `problem_number>=1` 的页数（排除姓名页 0）；`count_ok = (answer_pages == 题数 N)`；`total_pages==0` 的学生进 `absent_students`；`unlabeled_pages` = 本场 `student_id IS NULL` 的页数。

### fake.rs — 演示/测试播种
- `seed_fake_exam`：造「假奥赛物理」3 题 5 人，且某 (学生,题号) 挂 2 张溢出页，让轴 2 速览「有东西可看」。

## 命令层（IPC）

`AppState = Mutex<Option<OpenExam{ db, dir, exam_id }>>`——语义即「**当前打开的考试，或无**」。`with_exam(state, |oe| …)` 辅助：锁 → 取 `Some`（无则 `"no exam open"`）→ 在打开的考试上运行闭包，`anyhow::Error` 统一转 `String`（返回类型别名 `R<T>=Result<T,String>`）。需要同时做文件 IO 的命令（`ingest_folder`/`read_image`/`save_pdf_page`）手动内联同样的锁模式以拿到 `oe.dir`。

**app 自定义命令无需 capability 授权**（是本应用自己的代码）；**插件命令**（`tauri_plugin_dialog` 的开/存对话框）受 Tauri v2 权限模型管辖，必须在 `capabilities/default.json` 显式授予：`dialog:allow-open` + `dialog:allow-save`（`windows:["main"]`）。

**Tauri v2 参数映射**：Rust 侧 snake_case（`problem_id`），前端 `invoke` 传 camelCase（`problemId`），IPC 边界自动转换。下表用前端可见的 camelCase。

### 文件名安全护栏

`read_image` / `read_pdf` / `save_pdf_page` 对前端传入的 `filename` 统一拒绝含 `/`、`\`、`..`、`:` 的名字：

```rust
if filename.contains('/') || filename.contains('\\') || filename.contains("..") || filename.contains(':') {
    return Err("非法文件名".into());
}
```

- `/ \` 挡路径分隔符逃逸；`..` 挡上跳遍历。
- **`:` 在 Windows 上尤为关键**：Windows 有**盘符相对路径**（`C:foo` = C 盘当前目录下的 foo）和 NTFS 交替数据流（`file:stream`）。不禁 `:`，`Path::join` 遇到含盘符的串可能产生意外解析、逃出 `images/`。

## 前端结构

`App.vue` 是壳：顶栏 4 按钮切 `route`，`<main>` 单挂一个视图；`n-config-provider` 套 `darkTheme`，`.app-main` 给原生元素兜底浅色文字（naive-ui 暗主题只管自己的组件），`@media print` 隐藏顶栏。`api.ts` = 按阶段分组的 typed `invoke` 包装；`types.ts` = 与 Rust 对应的 TS 接口。

### 纯键盘 reducer 模式

`useGradeKeys.ts` / `useLabelKeys.ts` 是**无副作用纯函数** `reduce*(state, key, ctx) → { state, effect }`：只算新状态 + 一个**待执行 effect 描述**，绝不碰 IPC/DOM。真正的落库、翻图、刷新由视图的 `applyEffect` 执行。因此键位逻辑可被 Vitest 纯单测（无需挂组件、无需 mock Tauri）。

- `GradeState = { index, peek, manual, buffer, overview }`；`GradeEffect = none | setPreset | setManual | advance | back | flag | nextFlag | jump`。
- 视图 `onKey` 先拦截浮层类键（`?`/`R`/浮层 Esc/`Tab`），再交 reducer；`handled = effect≠none || 状态变了` 才 `preventDefault`（只吞真正消费的键）。

完整键位表见 [使用手册](03-使用手册.md#判分页快捷键)。判分页界面构成：顶栏（题目切换 + 当前份身份 + 分段进度条 + 下一个未判）、中间卷面（缩放/平移）、右侧极简面板（记分卡置顶 + 档位键 + 评语）、左侧可折叠悬浮面板 `GradeSidebar`（本题平均分 + 判分分布直方图 + 评分标准截断预览）、`R` 全文浮层、`?` 快捷键浮层、`G` 队列总览 modal。评分标准用 `marked` 渲染 Markdown。
